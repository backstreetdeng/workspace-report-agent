### 19:15 SSE closed file 报错修复

#### 现象
- 前端 SSE 机会分析报错：`I/O operation on closed file`。
- 复测问题：`分析15-20万新能源SUV市场机会`。

#### 根因
- `strategy-orchestrator` 的 `web-search` 通过 `importlib` 加载 `skills/tavily-search/scripts/tavily_search.py`。
- 该脚本在 import 时直接重写全局 `sys.stdout/sys.stderr`，这是 CLI 脚本副作用；在 Uvicorn/SSE 长进程中会污染输出流，最终可能导致 closed file 错误。

#### 修复
- `skills/tavily-search/scripts/tavily_search.py`
  - 将 Windows UTF-8 stdout/stderr 包装移入 `ensure_utf8_stdio()`。
  - 只在 CLI `main()` 中调用，作为模块 import 时不再修改宿主进程全局 stdio。
- `python_wrapper/live_agent_server.py`
  - 增加 `ANALYSIS_TIMEOUT_SECONDS=90`。
  - `/analyze` 使用 `asyncio.wait_for` 做超时保护。
  - `/analyze_sse` 超时返回明确 `TimeoutError` 事件。
  - SSE 异常写入 `temp/live_agent_server_runtime_error.log`，并在 error 事件返回 `error_type` 和 traceback tail，避免前端只看到模糊错误。
- 新增 `tests/test_tavily_import_safety.py`
  - 验证 import Tavily helper 不会替换 `sys.stdout/sys.stderr`。
